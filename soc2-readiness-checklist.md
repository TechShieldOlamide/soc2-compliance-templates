---

## 1. Security

- [ ] Access control policy documented and enforced
- [ ] Multi-factor authentication enabled on all critical systems
- [ ] Role based access control implemented
- [ ] Access reviews conducted quarterly
- [ ] Offboarding process removes access within 24 hours
- [ ] Firewall and network security controls documented
- [ ] Vulnerability scanning conducted regularly
- [ ] Penetration testing conducted at least annually
- [ ] Security incident response plan documented and tested
- [ ] Endpoint protection deployed on all devices
- [ ] Encryption at rest implemented for sensitive data
- [ ] Encryption in transit enforced (TLS 1.2 minimum)
- [ ] Security awareness training conducted for all staff
- [ ] Background checks conducted for employees with system access
- [ ] Change management process documented

---

## 2. Availability

- [ ] System uptime monitored and tracked
- [ ] Service level agreements documented
- [ ] Disaster recovery plan documented
- [ ] Business continuity plan documented and tested
- [ ] Backup procedures documented and tested regularly
- [ ] Capacity planning process in place
- [ ] Incident management process for availability issues
- [ ] Redundancy built into critical infrastructure

---

## 3. Processing Integrity

- [ ] Data processing accuracy controls documented
- [ ] Input validation implemented across systems
- [ ] Error handling and logging in place
- [ ] Quality assurance process for system changes
- [ ] Data reconciliation processes documented
- [ ] System monitoring for processing errors

---

## 4. Confidentiality

- [ ] Data classification policy documented
- [ ] Confidential data identified and labelled
- [ ] Access to confidential data restricted appropriately
- [ ] Non-disclosure agreements in place with employees and contractors
- [ ] Secure disposal process for confidential data
- [ ] Confidentiality requirements included in vendor contracts

---

## 5. Privacy

- [ ] Privacy policy published and accurate
- [ ] Personal data inventory maintained
- [ ] Lawful basis for processing documented
- [ ] Data subject rights process documented
- [ ] Data retention policy documented and followed
- [ ] Privacy impact assessments conducted for high risk processing
- [ ] Third party data sharing agreements in place

---

## 6. Governance and Risk Management

- [ ] Information security policy documented and approved
- [ ] Risk assessment conducted and documented
- [ ] Risk register maintained and reviewed regularly
- [ ] Security roles and responsibilities defined
- [ ] Board or leadership oversight of security program documented
- [ ] Policies reviewed and updated annually

---

## 7. Vendor and Sub-processor Management

- [ ] All vendors and sub-processors identified
- [ ] Vendor risk assessments conducted before onboarding
- [ ] Data processing agreements in place with all vendors
- [ ] Vendor SOC 2 reports or equivalent reviewed annually
- [ ] Vendor access reviewed and limited appropriately

---

## 8. Evidence and Documentation

- [ ] Policy documents centrally stored and version controlled
- [ ] Evidence collection process documented
- [ ] Evidence owners assigned for each control
- [ ] Audit trail maintained for key system changes
- [ ] Logs retained according to defined retention period

---

## 9. Audit Preparation

- [ ] Auditor selected and engagement letter signed
- [ ] Observation period defined (for Type II)
- [ ] Control owners briefed on their responsibilities
- [ ] Evidence collection platform configured (e.g. Sprinto, Vanta, Drata)
- [ ] Internal readiness assessment completed before formal audit begins
- [ ] Gaps identified and remediated before audit start
